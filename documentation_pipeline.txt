## 1. Project Objective

Build a clean, annotated medical Q&A dataset in Moroccan Darija (Moroccan Arabic dialect) from raw CSV data, intended for fine-tuning an LLM to serve as a Moroccan medical assistant.

## 3. Processing Pipeline — Overview

Raw CSVs (22 categories) → Heuristic Cleaning (pandas) → ChatGPT API (gpt-4o-mini) + NER Annotation → Post-validation (Latin purge) → Final JSON in processed_json/


## 4. Pipeline Details

Script: process_dataset_chatgpt.py
Model: OpenAI gpt-4o-mini
Batch size: 20 items per request
Max items: 1000 per category (or max available if < 1000)
Temperature: 0.3 (deterministic responses)
Response format: json_object (OpenAI native JSON mode)

Steps:

1. Heuristic cleaning (is_valid()):
   - Remove phrases with < 3 words
   - Filter out predefined Arabic "noise" phrases
   - Use df.iloc[:, 0] and df.iloc[:, 1] instead of column names (to handle typos in CSV headers)

2. Input sanitization (sanitize_text()):
   - Remove control characters (\x00-\x1f)
   - Replace double quotes with single quotes
   - Remove isolated backslashes
   - Prevents 400 JSON parsing errors from OpenAI API

3. ChatGPT annotation:
   - Each batch of 20 Q&A pairs is sent to gpt-4o-mini
   - The model cleans, improves, and annotates each entry
   - Automatic retry (3 attempts) on API errors
   - Skip existing files for resume capability

4. Post-validation (post_validate()):
   - Remove non-dict elements (hallucinations)
   - Remove entries with empty question or answer
   - Remove entries still containing Latin characters [a-zA-ZÀ-ÿ]
   - Normalize entities structure if missing

5. Output: One JSON file per category saved in processed_json/


## 5. ChatGPT Prompt Used

You are a medical dataset annotation expert.

Your task:
- Clean the Darija question (make it natural, short, like a real Moroccan patient speaks)
- Improve the answer: clear, max 3 sentences, proper punctuation, in Darija marocaine (Arabic script ONLY)
- Extract medical entities (NER)
- If the input has no medical sense, skip it entirely

CRITICAL RULES:
- ALL text must be in Arabic script ONLY. NO French, NO English words.
- If a medication name is in Latin characters, write it phonetically in Arabic (e.g., Paracetamol → باراسيتامول)

Return JSON: { category, question_darija, answer, entities: { symptoms, medical_field, urgency } }
Urgency values: منخفض (low), متوسط (medium), عالي (high), طارئ (emergency)


## 6. Final JSON Schema

Each entry in the output JSON files follows this structure:

{
  "category": "Cardiology",
  "question_darija": "الكفاءة ديال القلب 38%، شنو ندير؟",
  "answer": "هاد النسبة منخفضة وخصك تستاشر طبيب. يمكن يكون عندك مشكل فالقلب.",
  "entities": {
    "symptoms": [],
    "medical_field": "القلب",
    "urgency": "متوسط"
  }
}

| Field                  | Type         | Description                                              |
|------------------------|--------------|----------------------------------------------------------|
| category               | string       | Medical specialty (in English)                           |
| question_darija        | string       | Patient question in Darija (Arabic script only)          |
| answer                 | string       | Medical answer in Darija (2-3 sentences max)             |
| entities.symptoms      | list[string] | Extracted symptoms in Arabic                             |
| entities.medical_field | string       | Medical domain in Arabic                                 |
| entities.urgency       | string       | منخفض / متوسط / عالي / طارئ                              |

## 8. Technical Decisions

1. GPT-4o-mini: Chosen for reliable JSON output at a modest cost. Temperature set to 0.3 for deterministic, consistent results.
2. 1000 examples per category: Categories with fewer than 1000 valid entries use all available data.
3. Built-in NER: Each entry is enriched with extracted medical entities (symptoms, medical field, urgency level), enabling direct use for supervised fine-tuning or semantic search.
4. Strict post-validation: No Latin residue is tolerated in the final dataset. Any entry containing [a-zA-Z] characters is automatically removed.
5. Input sanitization: Control characters, double quotes, and backslashes are cleaned before sending to the API to prevent 400 JSON parsing errors.
6. Resume capability: The script skips categories that already have a JSON file in processed_json/, allowing safe restarts without reprocessing.
